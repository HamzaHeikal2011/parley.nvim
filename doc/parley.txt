INTRODUCTION                                          *parley*

A Neovim plugin that provides a persistent side-panel chat interface
for Ollama local LLM models. Inspired by Cursor's side panel, Claude Code's
VS Code extension, and Windsurf's context chips.

Features:
- Persistent right-side panel (not a floating window)
- Visual selection auto-attachment as context chips
- Streaming responses with markdown rendering
- Inline code block actions: Apply, Copy, Diff
- Per-buffer conversation sessions
- Zero plugin dependencies (only requires curl + Neovim 0.10+)
- Works out of the box, no setup() required

==============================================================================
REQUIREMENTS                                        *parley-requirements*

- Neovim >= 0.10
- curl (Ollama users typically have this)
- Ollama server running locally

==============================================================================
INSTALLATION                                        *parley-installation*

Using lazy.nvim:
>lua
  {
    "yourname/parley.nvim",
    event = "VeryLazy",
    opts = {
      url = "http://127.0.0.1:11434",
      model = "qwen2.5-coder:7b",
    },
    keys = {
      { "<leader>oc", "<cmd>Parley<cr>", desc = "Parley" },
      { "<leader>oa", "<cmd>ParleyAttach<cr>", desc = "Attach selection", mode = "v" },
      { "<leader>ob", "<cmd>ParleyAttachBuffer<cr>", desc = "Attach buffer" },
      { "<leader>ox", "<cmd>ParleyClearContext<cr>", desc = "Clear context" },
      { "<leader>om", "<cmd>ParleyModel<cr>", desc = "Switch model" },
    },
  }
<

==============================================================================
COMMANDS                                            *parley-commands*

:Parley [subcommand]                            *:Parley*
    Main command. Subcommands:
    - open (default) — Open the chat panel
    - close           — Close the chat panel
    - toggle          — Toggle the chat panel
    - input           — Open the input area
    - stop            — Stop generation
    - clear           — Clear conversation
    - model           — Switch model
    - attach          — Attach selection/buffer
    - context_clear   — Clear context chips

:ParleyAttach                                   *:ParleyAttach*
    Attach current visual selection as a context chip.

:ParleyAttachBuffer                             *:ParleyAttachBuffer*
    Attach the entire current buffer as a context chip.

:ParleyClearContext                             *:ParleyClearContext*
    Remove all context chips.

:ParleyModel                                    *:ParleyModel*
    Select a different Ollama model.

:ParleyStop                                     *:ParleyStop*
    Stop the current generation.

:ParleyClear                                    *:ParleyClear*
    Clear the conversation history.

==============================================================================
KEYMAPS                                             *parley-keymaps*

Default keymaps (in normal mode unless specified):

<leader>oc          Toggle chat panel
<leader>oa          Attach visual selection (visual mode)
<leader>ob          Attach entire buffer
<leader>ox          Clear context chips
<leader>om          Switch model

Inside the chat panel:
q                   Close panel
<C-l>               Clear conversation
<C-c>               Stop generation
i or a              Open input area
<C-y>               Apply code block under cursor
<C-d>               Copy code block under cursor
<C-f>               Show diff for code block under cursor

Inside the input area:
<CR>                Submit (single line) or new line (multi-line)
<C-CR>              Always submit
<Esc>               Close input
<C-c>               Stop generation

==============================================================================
PLUG MAPPINGS                                       *parley-plug*

<Plug>(Parley)              Toggle chat panel
<Plug>(ParleyAttach)        Attach selection (visual mode)
<Plug>(ParleyAttachBuffer)  Attach buffer
<Plug>(ParleyClearContext)  Clear context
<Plug>(ParleyModel)         Switch model

Example custom mapping:
>lua
  vim.keymap.set("n", "<C-s>", "<Plug>(Parley)")
  vim.keymap.set("v", "<C-s>", "<Plug>(ParleyAttach)")
<

==============================================================================
WORKFLOW                                            *parley-workflow*

1. Select code in visual mode
2. Press <leader>oa to attach selection (shows as a chip)
3. Press <leader>oc to open the chat panel
4. Press 'i' to open the input area
5. Type your question
6. Press <CR> (single line) or <C-CR> (multi-line) to send
7. Response streams in the panel
8. Navigate to a code block and press:
   - <C-y> to Apply the code
   - <C-d> to Copy to clipboard
   - <C-f> to show a Diff
9. Keep the conversation going with follow-up questions

==============================================================================
API                                                 *parley-api*

Lua API for programmatic use:
>lua
  local chat = require("parley")

  chat.toggle()                     -- Toggle panel
  chat.attach_selection()           -- Attach visual selection
  chat.attach_buffer()              -- Attach buffer
  chat.send_message("Explain this") -- Send a message
  chat.stop()                       -- Stop generation
  chat.clear()                      -- Clear conversation
  chat.switch_model()               -- Switch model
  chat.status()                     -- "IDLE" or "WORKING"
<
